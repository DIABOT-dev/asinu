2. Cài Đặt Công Cụ
2.1. Node.js & npm
# Dùng nvm (khuyến nghị)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 20
nvm use 20

# Hoặc tải từ https://nodejs.org
2.2. Android Studio & SDK
Tải Android Studio từ https://developer.android.com/studio
Mở Android Studio → Settings → SDK Manager
Cài đặt các component:
✅ Android SDK Platform 35
✅ Android SDK Build-Tools 35.0.0
✅ NDK (Side by side) - version 26.1.10909125
✅ Android SDK Command-line Tools
✅ Android Emulator
✅ Android SDK Platform-Tools
✅ CMake (nếu cần native modules)
Cấu hình biến môi trường — thêm vào ~/.zshrc hoặc ~/.bash_profile:
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/platform-tools
export PATH=$PATH:$ANDROID_HOME/tools
export PATH=$PATH:$ANDROID_HOME/tools/bin

# QUAN TRỌNG: Java 17 cho React Native 0.76+
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
# Apply
source ~/.zshrc
Kiểm tra:
echo $ANDROID_HOME    # Phải ra path đến SDK
java -version         # Phải là 17.x
adb devices           # Phải chạy được

3. Setup Project Lần Đầu
3.1. Clone & Install
cd /path/to/project
cd asinu

# Cài dependencies
npm install

# Copy env file
cp .env.example .env
# → Sửa .env theo môi trường của bạn
3.2. Cấu hình .env
Các biến quan trọng cần set:

# API backend
EXPO_PUBLIC_API_BASE_URL=https://asinu.top

# OAuth (nếu cần test đăng nhập thật)
EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID=...
EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID=...
EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=...

# Development mode
EXPO_PUBLIC_USE_MOCK_OAUTH=true   # true nếu chưa setup OAuth
EXPO_PUBLIC_DEV_BYPASS_AUTH=0
3.3. Prebuild Native Projects
# Sinh lại thư mục android/ và ios/ từ app.json
npx expo prebuild

# Nếu muốn prebuild sạch (xóa cũ, tạo mới)
npx expo prebuild --clean
⚠️ LƯU Ý: Nếu đã có thư mục android/ và ios/ sẵn trong repo, KHÔNG chạy prebuild --clean trừ khi bạn biết mình đang làm gì. Nó sẽ xóa toàn bộ native code tùy chỉnh.

3.4. Install iOS Pods
cd ios
pod install
cd ..

# Nếu lỗi, thử:
cd ios
pod deintegrate
pod install --repo-update
cd ..
4. Build Android
4.1. Chạy Debug trên Emulator/Device
# Cách 1: Qua Expo
npx expo run:android

# Cách 2: Qua npm script
npm run android

# Cách 3: Trực tiếp Gradle
cd android
./gradlew assembleDebug
4.2. Build Release APK
# Script có sẵn
npm run build:android

# Hoặc trực tiếp
cd android
./gradlew assembleRelease

# APK output tại:
# android/app/build/outputs/apk/release/app-release.apk
4.3. Build Release AAB (cho Google Play)
cd android
./gradlew bundleRelease

# AAB output tại:
# android/app/build/outputs/bundle/release/app-release.aab
4.4. Cấu hình Signing (Release)
Tạo file android/app/signing.properties hoặc set trong android/gradle.properties:

# Tạo keystore trước:
# keytool -genkeypair -v -storetype PKCS12 -keystore asinu-release.keystore -alias asinu -keyalg RSA -keysize 2048 -validity 10000

MYAPP_UPLOAD_STORE_FILE=asinu-release.keystore
MYAPP_UPLOAD_KEY_ALIAS=asinu
MYAPP_UPLOAD_STORE_PASSWORD=***
MYAPP_UPLOAD_KEY_PASSWORD=***
🔴 QUAN TRỌNG: KHÔNG BAO GIỜ commit keystore và password lên Git!

5. Build iOS
5.1. Chạy Debug trên Simulator
# Cách 1: Qua Expo
npx expo run:ios

# Cách 2: Qua npm script  
npm run ios

# Chọn simulator cụ thể
npx expo run:ios --device "iPhone 16 Pro"
5.2. Chạy trên Device thật
Kết nối iPhone qua USB
Mở Xcode → mở ios/Asinu.xcworkspace (KHÔNG mở .xcodeproj)
Chọn Team trong Signing & Capabilities
Chọn device thật làm target
Build & Run (⌘+R)
Hoặc qua CLI:

npx expo run:ios --device
# Chọn device từ danh sách
5.3. Build IPA (Release)
# Qua Xcode:
# 1. Mở ios/Asinu.xcworkspace
# 2. Product → Scheme → Edit Scheme → Set Release
# 3. Product → Archive
# 4. Organizer → Distribute App

# Hoặc qua xcodebuild:
cd ios
xcodebuild -workspace Asinu.xcworkspace \
  -scheme Asinu \
  -sdk iphoneos \
  -configuration Release \
  archive -archivePath build/Asinu.xcarchive

7. Lỗi Thường Gặp & Cách Sửa
🔴 Android
Lỗi	Nguyên nhân	Cách sửa
SDK location not found	Chưa set ANDROID_HOME	Thêm vào ~/.zshrc (xem mục 2.2)
Could not determine java version	Sai JDK	Cài JDK 17, set JAVA_HOME
Execution failed for task ':app:mergeDebugNativeLibs'	NDK thiếu	Cài NDK 26.1.10909125 qua SDK Manager
Build failed with an exception - Gradle	Cache hỏng	cd android && ./gradlew clean
Unable to load script	Metro chưa chạy	Chạy npx expo start trước
java.lang.OutOfMemoryError	RAM Gradle thiếu	Đã set -Xmx2048m trong gradle.properties
Duplicate class	Dependency conflict	cd android && ./gradlew clean rồi rebuild
INSTALL_FAILED_UPDATE_INCOMPATIBLE	Signature mismatch	Gỡ app cũ trên device rồi cài lại
Reset Android build hoàn toàn:

cd android
./gradlew clean
rm -rf .gradle
rm -rf app/build
cd ..
npm install
npx expo run:android
🔴 iOS
Lỗi	Nguyên nhân	Cách sửa
No bundle URL present	Metro chưa chạy	Chạy npx expo start trước
Pod install failed	Cache Pods hỏng	Xem “Reset iOS” bên dưới
Signing requires a development team	Chưa chọn Team	Mở Xcode → Signing → chọn Team
The sandbox is not in sync	Pods outdated	cd ios && pod install
Framework not found hermes	Hermes config sai	Kiểm tra Podfile.properties.json
Build input file cannot be found	File reference hỏng	Clean build folder (⌘+Shift+K)
Multiple commands produce	Duplicate resources	Clean + pod deintegrate + pod install
Undefined symbol (linking)	Missing pod	pod install --repo-update
Reset iOS build hoàn toàn:

cd ios
pod deintegrate
rm -rf Pods
rm -rf build
rm Podfile.lock
pod install --repo-update
cd ..

8. Lưu Ý Quan Trọng
⚡ Performance
Hermes đang được bật cho iOS (Podfile.properties.json: hermes), Android config là jsc trong gradle.properties → kiểm tra lại nếu muốn đồng nhất
New Architecture đang enabled trên iOS, disabled trên Android (newArchEnabled=false trong gradle.properties) → Có thể gây khác biệt behavior giữa 2 platform
Gradle JVM đã set Xmx2048m — nếu máy yếu có thể bị chậm, máy mạnh tăng lên 4096m
🔒 Bảo Mật
KHÔNG commit file .env lên Git (đã có trong .gitignore)
KHÔNG commit keystore / signing certificates
KHÔNG để OAuth Client Secret trong code frontend
Dùng expo-secure-store cho sensitive data (đã cài)
Google OAuth URL Scheme đã cấu hình trong app.json → giữ nguyên
📦 Dependencies Đặc Biệt
Project sử dụng patch-package cho 3 packages:

expo-application+7.0.8
expo-crypto+15.0.8
expo-web-browser+15.0.10
→ Sau mỗi lần npm install, patches tự động apply. Nếu upgrade các package này, phải tạo lại patch hoặc kiểm tra compatibility.

# 3. Xóa iOS build

rm -rf ios/build
rm -rf ~/Library/Developer/Xcode/DerivedData/Asinu-*

# 4. Xóa Metro & Expo cache
rm -rf .expo
